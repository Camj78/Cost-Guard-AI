# Safety Score v1.0.0

**analysis_version:** 1.0.0
**release_date:** 2026-03-09
**spec_version:** v1.0

---

## Changes

Initial scoring release. Establishes the CostGuard Safety Score (CSS) as a deterministic,
heuristic-based measure of prompt security and operational reliability.

## Components

Five scoring components introduced:

| Component | Internal Bucket | Weight |
|---|---|---|
| Prompt Injection | structural | 20% |
| System Override | structural | 20% |
| Jailbreak Behavior | ambiguity | 20% |
| Token Cost Explosion | length + context + volatility | 60% combined |
| Tool Abuse | ambiguity + structural | 40% combined |

**Bucket weights:**

| Bucket | Weight |
|---|---|
| length | 25% |
| context | 20% |
| ambiguity | 20% |
| structural | 20% |
| volatility | 15% |

## Score Bands

| Range | Band |
|---|---|
| 91–100 | Hardened |
| 71–90 | Safe |
| 41–70 | Needs Hardening |
| 0–40 | Unsafe |

## Threat Intelligence Adjustments

CVE match adjustments introduced at v1.0.0:

| Severity | Risk Score Adjustment |
|---|---|
| Critical | +10 |
| High | +7 |
| Medium | +3 |

Adjustments are additive and bounded. Maximum CSS impact: −10 points per CVE match.
